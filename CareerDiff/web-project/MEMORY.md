# 메모리 — GPT/Claude 웹사이트 "메모리" 기능 사용 안내

> 이 문서는 로컬 코드가 아니라 **ChatGPT/Claude 웹사이트의 계정 전역 "메모리" 기능을 CareerDiff 테스트에 쓸지 판단하기 위한 안내 문서**입니다.

## 핵심 결론

CareerDiff 프롬프트 테스트에는 계정 전역 메모리 기능을 **사용하지 않는 것을 권장**합니다. `INSTRUCTIONS.md`(지침) + `KNOWLEDGE_FILES.md`(지식파일)만으로 매 대화에 필요한 컨텍스트가 전부 재현되기 때문입니다.

## 이유

- ChatGPT의 "Memory"는 프로젝트 범위가 아니라 **계정 전체**에 남습니다. CareerDiff 테스트용으로 넣은 내용이 다른 무관한 대화에도 노출됩니다.
- Claude Projects는 프로젝트 단위 지침/지식만 있고, 별도의 전역 "메모리" 개념이 없습니다(2026-07 기준). 즉 Claude에서는 이 문서가 해당 없음 — Instructions/Knowledge만 신경 쓰면 됩니다.
- 테스트 재현성 관점에서도, 매 실행이 지침+지식파일만으로 결정되어야 "같은 입력 → 같은 결과"를 비교하기 쉽습니다. 메모리에 내용이 섞이면 이전 대화의 잔여 컨텍스트가 결과에 영향을 줘 재현이 어려워집니다.

## 예외적으로 메모리에 남겨도 되는 경우

다음처럼 **CareerDiff에 국한되지 않는, 사용자 개인의 일반적인 선호**라면 메모리에 남겨도 무방합니다.

- "항상 한국어로 답변" 같은 언어 선호
- "코드/JSON 예시는 항상 마크다운 코드블록으로" 같은 서식 선호

## 메모리에 절대 넣지 말 것

- 테스트용이라도 합성이 아닌 실제 이력서·채용공고 원문
- CareerDiff 분석 규칙 자체(이건 지침 칸/지식파일에 있어야 매 대화 재현 가능)
- API 키, 토큰, 내부 식별자
