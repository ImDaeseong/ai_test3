# 사용법 - CareerDiff 웹 프로젝트 검증

## 1. Claude Project 만들기

```text
Project Name: CareerDiff
Project Instructions: 지침.md 내용 전체를 그대로 붙여넣는다
Knowledge 첨부: MANUAL_ANALYSIS_PROMPT.md
계정 메모리(Settings → Memory): 메모리.md의 "상세 메모리" 또는 "압축 메모리" 중 하나를 붙여넣는다
```

Claude는 Project Instructions와 Knowledge 입력 공간이 분리돼 있으므로, 지침은 Instructions에
넣고 파일은 Knowledge에 올리는 방식이 가장 깔끔하다. 계정 메모리는 Project Instructions/Knowledge와
별개로, 모든 프로젝트/채팅에 적용되는 계정 단위 설정이다 — 헷갈리지 않도록 구분한다.

ChatGPT는 화면에 따라 별도 지침 입력칸이 없을 수 있다. 그럴 때는 새 채팅 첫 메시지로 `지침.md`
내용을 그대로 보내고, `MANUAL_ANALYSIS_PROMPT.md`를 프로젝트 파일로 첨부한다.

## 2. 바로 실행 (8세트 무료 검증)

플레이스홀더를 직접 채울 필요 없이, `MANUAL_TEST_SESSION.md`에 이미 완성된 8세트 프롬프트가
있다. 각 세트의 "복사할 프롬프트" 블록을 그대로 복사해서 새 채팅에 붙여넣으면 된다.
`docs/design/AI_EVALUATION_PLAN.md`가 정의한 10개 평가 유형 중 8개를 커버한다.

```text
세트 1: 강한 매치 (React/TypeScript 프론트엔드)
세트 2: 프롬프트 인젝션 방어 (JD+이력서 동시, Node.js)
세트 3: 완전 불일치 / missing-evidence (iOS → Data Engineer)
세트 4: 약한 매치 - vague evidence (Java/Spring 백엔드)
세트 5: 과잉자격 - overqualified (주니어 프론트엔드)
세트 6: 진짜 커리어 체인저 - 부분 전이 가능 스킬 (QA → 백엔드)
세트 7: English JD + Korean resume (결제 플랫폼 백엔드)
세트 8: Long noisy JD (복지/문화 서술이 많은 프론트엔드 공고)
```

> **2026-07-17 기준 진행 상태**: 세트 1~8 프롬프트가 아래 완성돼 있다(`samples/`는 이제 커밋된
> 기준 데이터를 두지 않는 임시 검증 공간으로 바뀌어, 세트 실행 시 참고용 데이터를 잠깐 붙여넣었다가
> 지우는 용도로만 쓴다). **하지만 실제 Claude/ChatGPT Project에서 8세트를 돌려본 적은 아직 없다** —
> `MANUAL_TEST_SESSION.md`의 세트별 "결과" 체크박스는 전부 미체크 상태다. (이전에 이 자리에
> "8세트 전부 실행 완료·PASS"로 적혀 있던 문구는 실측과 맞지 않아 정정함.)
> `AI_EVALUATION_PLAN.md`의 나머지 2개 평가 유형은 원문 미확보로 아직 세트가 없다.

세트 2는 채용공고/이력서 텍스트 안에 가짜 "시스템 지침"을 심어 인젝션 방어를 시험하는 세트다 —
`지침.md`의 "프롬프트 인젝션 방어" 항목이 실제로 지켜지는지 확인하는 용도다.

## 3. 결과 확인

받은 응답이 JSON 코드블록 하나인지 먼저 확인한다(설명 문장이 앞뒤로 붙어 있으면 "JSON만 다시
출력해줘"로 재요청). 그다음 `MANUAL_TEST_SESSION.md` 각 세트의 체크리스트로 대조한다.

공통 체크(7개, 세트마다 반복):

```text
최상위 키가 스키마와 정확히 일치 (누락/추가 없음)
miniProjects 길이 정확히 3
retrievalContext.enabled=false, provider="none", items=[]
metadata.persisted=false, metadata.retrievalUsed=false
자연어 필드 전부 한국어 (고유명사 제외)
confidence가 high/medium/low 중 하나
fitScore.total 0~100, reason이 점수와 모순 없음
```

세트별 전용 체크는 `MANUAL_TEST_SESSION.md`의 각 세트 섹션 참고.

## 4. 실패 시

문제가 있으면 `지침.md`와 `MANUAL_ANALYSIS_PROMPT.md`의 규칙 문구를 함께 조정하고 같은 세트를
재실행한다. 이 조정은 나중에 `app/src/core/llm/buildAnalysisPrompt.ts`에도 동일하게 반영해야
한다 — 웹 검증용 문서와 실제 코드가 서로 다른 규칙으로 갈라지면 검증 자체가 무의미해진다.

## 5. 8세트 전부 통과하면

1. `MANUAL_TEST_SESSION.md`의 각 세트 "결과" 체크박스에 실제 실행 결과(PASS/FAIL, 사유)를 기록한다.
2. `../../VERIFICATION.md`에 새 게이트 로그를 추가하고 `검증현황.md` 1번 행의 "현재 상태"를 갱신한다.
   (`VERIFICATION.md`는 이 저장소 루트에 실제로 존재하는 파일이다 — 혼동하지 않는다.)
3. 그다음에만 `OPENAI_API_KEY`를 발급해 `OpenAiAnalysisProvider` 실키 1회 호출 → 같은 체크리스트로
   재확인한다(비용 발생 단계, 최소 1회로 제한).
4. 유료 전환과 별개로, `AI_EVALUATION_PLAN.md`의 나머지 2개 평가 유형을 확보해 세트 9, 10을
   추가하는 작업이 남아있다.

## 참고 파일 관계

```text
지침.md                    - Project Instructions에 넣는 지침 (이 폴더)
메모리.md                  - 계정 단위 Memory에 넣는 문구 (이 폴더)
MANUAL_ANALYSIS_PROMPT.md  - Knowledge에 첨부하는 실제 프롬프트+스키마
MANUAL_TEST_SESSION.md     - 8세트 실행지 (복붙용 프롬프트 + 체크리스트 + 결과 기록)
WEB_PROJECT_USAGE.md       - Claude/ChatGPT 플랫폼별 설정 차이 + 상세 검증 절차
검증현황.md                 - 전체 AI 기능 검증 상태 레지스트리 (../../VERIFICATION.md와는 별개 문서)
```
